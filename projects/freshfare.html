FreshFare

Java Point-of-Sale System

Screenshots

Project Overview

FreshFare is a desktop-based Point of Sale (PoS) system developed using Java Swing. As the first major software project I worked on during my first year at university, it introduced me to object-oriented programming and desktop application development. The system includes features for processing sales, managing product inventory, and storing data using Java File I/O, providing hands-on experience in building a complete desktop application without relying on external frameworks or databases.

My Contributions

Developed as a four-member team project. My primary contributions included:

  • Designed the main POS workflow and screen flow
  • Built the core GUI screens for sales and inventory
  • Implemented transaction processing and inventory updates
  • Managed data persistence using Java File I/O
  • Tested the app end-to-end for a complete desktop experience

Technical Implementation

FreshFare follows an object-oriented design using Java Swing to create a responsive desktop interface. Product information and transaction data are stored in flat text files through Java File I/O (FileReader, BufferedReader, and FileWriter), allowing inventory records to persist between application sessions. User interactions are handled through Swing event listeners, enabling real-time inventory updates and transaction processing without the need for an external database.
